# Kids Caller — Notificação de Chamada de Crianças

## 1. Overview

Web app minimalista para igrejas onde o ministério infantil envia o número de uma criança e o operador de som recebe uma notificação instantânea. Duas telas: um PIN pad (sender) e um painel de notificações (receiver). Áreas afetadas: frontend (Vite + vanilla TS), banco de dados (Supabase), deploy (Cloudflare Pages).

**User story:** "Como auxiliar do ministério infantil, quero digitar rapidamente o número de uma criança num tablet, para que o operador de som receba a notificação na hora e chame os pais."

**Stack:**
- Frontend: Vite + Vanilla TypeScript
- Backend: Supabase (Realtime Database)
- Notificações: Web Notifications API + áudio
- Deploy: Cloudflare Pages (free tier)

**Requisitos operacionais críticos:**
- A página deve permanecer ativa indefinidamente — sem session expiry, sem logout, sem bloqueio de tela
- A conexão Realtime do Supabase deve reconectar automaticamente se cair, sem intervenção do usuário
- Push notifications devem funcionar de forma confiável mesmo com a aba em segundo plano
- Usar Wake Lock API para impedir que o dispositivo durma/bloqueie

### Phase 5: Feedback do Receiver + Vibração

**Goal:** O receiver pode confirmar recebimento de um chamado, enviando um alerta visual ao sender. Além disso, o celular do receiver vibra ao receber um novo chamado.

**What to do:**

#### 5a. Vibração ao receber chamado
- Ao receber novo chamado no receiver, chamar `navigator.vibrate([200, 100, 200])` (vibra 200ms, pausa 100ms, vibra 200ms)
- Feature-detect: só chamar se `'vibrate' in navigator` for true (desktop ignora silenciosamente)

#### 5b. Coluna `acknowledged` no banco
- Adicionar coluna `acknowledged` (boolean, default false) na tabela `chamados` no Supabase
- Atualizar a interface `Chamado` no receiver.ts para incluir `acknowledged: boolean`

#### 5c. Botão de feedback no receiver
- Em cada card de chamado NÃO lido, adicionar um botão "✓ Confirmar" (ou ícone de check) ao lado do número
- Ao clicar no botão de feedback:
  - Atualizar o registro no Supabase: `UPDATE chamados SET acknowledged = true WHERE id = X`
  - Marcar também como `read = true`
  - Renderizar o card com estado visual de "confirmado" (ex: ícone de check verde, texto "Confirmado")
- O botão de feedback desaparece após confirmação (evitar duplo clique)

#### 5d. Sender escuta feedback em tempo real
- No sender, após enviar um chamado com sucesso, assinar o canal Realtime do Supabase para escutar UPDATE na tabela `chamados` filtrando pelo `id` do chamado recém-enviado
- Quando o UPDATE com `acknowledged = true` chegar:
  - Mostrar alerta visual no display do PIN pad (ex: "Chamado #42 recebido! ✓") com cor verde, por ~3 segundos
  - Tocar som curto de confirmação (tom diferente do alerta)
  - Se o sender já enviou outro número, mostrar o feedback como toast/banner no topo sem interromper digitação
- Cancelar subscription quando o sender navegar para outra tela (cleanup)

#### 5e. Tratamento de múltiplos chamados pendentes
- O sender pode enviar vários chamados sem esperar feedback — manter lista interna dos IDs enviados na sessão
- Escutar UPDATE para todos os IDs pendentes (filtro `in` ou subscription genérica com filtragem client-side)
- Remover o ID da lista de pendentes quando feedback chegar

## 3. Edge Cases

- Se a conexão cair entre sender e Supabase, mostrar mensagem "Sem conexão" e impedir envio até reconectar.
- Se o receiver perder conexão WebSocket, reconectar automaticamente e buscar chamados perdidos desde o último recebido. Mostrar indicador visual do status da conexão.
- Se o Wake Lock for liberado pelo browser (ex: bateria baixa), tentar reativar automaticamente quando a página voltar ao foco.
- Se o usuário negar permissão de notificação, mostrar aviso persistente no receiver explicando que não receberá alertas em segundo plano.
- Se o browser não suportar Wake Lock API, mostrar aviso sugerindo desativar bloqueio de tela manualmente nas configurações do dispositivo.
- Se dois senders enviarem ao mesmo tempo, ambos devem chegar como registros separados (Supabase lida com isso nativamente).
- Se o browser não suportar Web Notifications (raro), degradar gracefully — só som + visual.
- Se o receiver confirmar um chamado mas a conexão do sender caiu, o sender não verá o feedback — mas ao reconectar, não precisa buscar feedbacks antigos (fire-and-forget, o feedback é efêmero).
- Se o browser não suportar Vibration API (desktop, iOS Safari), degradar silenciosamente — só som + notificação visual.
- Se o receiver clicar "Confirmar" duas vezes rapidamente, ignorar o segundo clique (botão já removido após primeiro).
